\chapter{Analyse et Étude Préalable}

Ce premier chapitre a pour objectif de poser les bases du projet. Il définit le cadre du travail, décrit les besoins des utilisateurs et définit les caractéristiques techniques et fonctionnelles.

\section{État de l'art}
L'application de l'intelligence artificielle au football n'est pas nouvelle, mais ses méthodes ont beaucoup évolué.

Historiquement, les modèles de prédiction utilisaient la loi de Poisson pour calculer la distribution des buts \cite{maher1982}. Bien qu'efficaces, ces modèles ont un problème majeur : ils supposent que les buts marqués par chaque équipe sont des événements indépendants, ce qui est contredit par la réalité d'un match (un but change la stratégie adverse). Dixon et Coles \cite{dixon1997} ont proposé une correction de cette indépendance pour les petits scores (0-0, 1-0, etc.), améliorant les prédictions.

Récemment, l'utilisation des méthodes d'apprentissage automatique a transformé ce domaine, permettant d'utiliser des variables complexes (fatigue, forme récente, expected goals). Afin d'expliquer notre choix technique, le tableau \ref{tab:comparatif_modeles} présente un résumé des différents modèles de classification habituellement utilisés dans les études existantes \cite{bunker2019}.

Face à ces alternatives, notre choix s'est porté sur une méthode plus performante pour les données tabulaires : le \textbf{Gradient Boosting}, et plus particulièrement son implémentation \textbf{XGBoost} \cite{chen2016xgboost}. Contrairement au Random Forest qui construit ses arbres séparément, XGBoost les construit l'un après l'autre, chaque nouvel arbre corrigeant les erreurs des précédents. Cette approche offre une grande souplesse et des performances souvent supérieures aux modèles traditionnels.

\section{Cahier des charges fonctionnel}
L'application doit proposer un ensemble de fonctionnalités selon le type d'utilisateur.

\subsection{Fonctionnalités pour l'Utilisateur Standard}
\begin{itemize}
    \item \textbf{Consultation des matchs} : Voir le calendrier des matchs à venir et les résultats récents.
    \item \textbf{Analyse des compositions} : Consulter les formations (ex : 4-3-3), les titulaires et les remplaçants.
    \item \textbf{Demande de prédiction} : Obtenir l'estimation de l'IA (pourcentages de chances) pour un match précis.
    \item \textbf{Comparatif tactique} : Comparer visuellement (graphiques Radar) la forme et les statistiques des deux équipes.
\end{itemize}

\subsection{Fonctionnalités pour l'Administrateur}
\begin{itemize}
    \item \textbf{Gestion des accès} : Voir les utilisateurs inscrits sur la plateforme.
    \item \textbf{Mise à jour des données} : Lancer manuellement la récupération des données depuis l'API-Football (Matchs, Statistiques, Compositions).
\end{itemize}

\section{Exigences Non-Fonctionnelles}
En plus des fonctionnalités de base, le système doit répondre à des critères de qualité précis :
\begin{itemize}
    \item \textbf{Sécurité} : L'accès doit être protégé par une connexion sécurisée (JWT). Les mots de passe sont cryptés en base de données.
    \item \textbf{Qualité des données} : Le système garantit l'unicité des matchs importés via l'identifiant \texttt{fixture\_id} pour éviter les doublons.
    \item \textbf{Rapidité} : Le moteur d'IA doit fournir une réponse en moins de 0,5 seconde une fois les données chargées.
\end{itemize}

\section{Analyse des Risques}
Le projet identifie deux risques majeurs liés aux données externes :
\begin{itemize}
    \item \textbf{Limitation des API (Rate Limiting)} : L'API gratuite limite le nombre de requêtes par jour. Pour résoudre cela, nous enregistrons les données localement (SQLite) pour ne pas interroger l'API plusieurs fois pour le même match.
    \item \textbf{Qualité des données} : Certaines compositions peuvent être incomplètes. Le système a été conçu pour être souple et traiter ces cas sans bloquer l'utilisateur.
\end{itemize}
