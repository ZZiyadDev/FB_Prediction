\section{Fondements Théoriques de l'Intelligence Artificielle}

\subsection{Le Problème de Déséquilibre des Classes (\textit{Imbalanced Data})}
Dans le football de haut niveau, les résultats ne sont pas répartis de manière égale. Les victoires à domicile (\textit{Home Win}) représentent environ 45\% des cas, les victoires à l'extérieur (\textit{Away Win}) environ 30\%, et les matchs nuls (\textit{Draw}) environ 25\%.
Si nous entraînons un algorithme sur ces données sans modification, il aura tendance à privilégier l'équipe à domicile et ne réussira pas à prédire le match nul, car c'est le cas le moins fréquent.

\subsubsection{Approche Mathématique : SMOTE}
Pour résoudre ce problème, nous avons utilisé \textbf{SMOTE} (\textit{Synthetic Minority Over-sampling Technique}) \cite{chawla2002smote}. Contrairement à la simple copie de données, SMOTE crée des exemples synthétiques entre les points existants de la classe minoritaire.

Soit un point $x_i \in \text{Classe Minoritaire}$. On sélectionne l'un de ses $k$ plus proches voisins, $x_{zi}$. Le nouvel échantillon synthétique $x_{syn}$ est généré par la formule :
\begin{equation}
    x_{syn} = x_i + \lambda \times (x_{zi} - x_i)
\end{equation}
où $\lambda$ est un nombre aléatoire entre 0 et 1. Cette technique permet d'enrichir les données d'entraînement et force le modèle à mieux apprendre les caractéristiques du ``Match Nul''.

\subsection{Algorithme Prédictif : XGBoost}
Nous avons choisi \textit{Extreme Gradient Boosting} (XGBoost), un algorithme basé sur les arbres de décision \cite{chen2016xgboost}. Il utilise la technique du \textit{Boosting} où chaque nouvel arbre est construit pour corriger les erreurs des arbres précédents.

\subsubsection{Fonction d'Objectif et Régularisation}
XGBoost cherche à minimiser une fonction d'objectif composée d'une fonction de perte (\textit{Loss}) et d'un terme de régularisation (pour éviter de trop coller aux données d'entraînement) :
\begin{equation}
    \mathcal{L}^{(t)} = \sum_{i=1}^n l\left(y_i, \hat{y}_i^{(t-1)} + f_t(x_i)\right) + \Omega(f_t)
\end{equation}
Avec le terme de régularisation défini par :
\begin{equation}
    \Omega(f) = \gamma T + \frac{1}{2} \lambda \sum_{j=1}^T w_j^2
\end{equation}
où $T$ est le nombre de feuilles de l'arbre, et $w_j$ le poids de la feuille $j$.

\subsubsection{Adaptation à la classification multiclasse}
Puisque notre problème compte 3 issues possibles (H, D, A), le modèle utilise la fonction objectif \texttt{multi:softprob}. Il applique une fonction Softmax sur les scores de sortie pour générer une distribution de probabilité :
\begin{equation}
    P(y=k \mid x) = \frac{e^{z_k}}{\sum_{j=1}^K e^{z_j}}
\end{equation}
C'est cette probabilité (ex : Home 45\%, Draw 20\%, Away 35\%) qui est renvoyée au frontend React pour afficher le graphique.

\section{Ingénierie des Caractéristiques (\textit{Feature Engineering})}
L'algorithme utilise un vecteur de 25 caractéristiques. Pour assurer le bon fonctionnement du modèle, toutes les variables numériques sont normalisées avec \textbf{StandardScaler} :
\begin{equation}
    z = \frac{x - \mu}{\sigma}
\end{equation}
où $\mu$ est la moyenne et $\sigma$ l'écart-type. Parmi les variables les plus importantes :
\begin{itemize}
    \item \textbf{Form Points} : Points obtenus lors des 5 derniers matchs (3 pour une victoire, 1 pour un nul).
    \item \textbf{Goal Ratios} : Moyenne de buts marqués vs encaissés sur la saison.
    \item \textbf{Home Advantage} : Variable représentant l'avantage de jouer à domicile.
\end{itemize}

\section{Évaluation et Validation du Modèle}
Pour garantir la fiabilité de nos prédictions, le système compare toujours le \textit{Résultat Prédit} au \textit{Résultat Réel} une fois le match terminé.
\begin{itemize}
    \item \textbf{Accuracy Score} : Pourcentage total de bonnes prédictions.
    \item \textbf{Matrice de Confusion} : Analyse des erreurs pour voir quels types de résultats sont les plus difficiles à prédire.
\end{itemize}
Ces indicateurs sont calculés par le backend et affichés sur le tableau de bord de l'administrateur pour suivre la performance de l'IA.
